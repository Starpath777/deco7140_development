Variations from A3 Design Proposal

  • Discussion Feature Limitation


    In the original A3 proposal, we committed to a full-fledged threaded discussion system: users could create posts, reply to any comment, and see updates in real time. Deviation: Due to the absence of a backend discussion-thread API, time constraints, and my current experience level, we simulated this feature entirely on the frontend. New posts are stored in browser memory (sessionStorage) and rendered immediately, but replies are not persisted—refreshing the page clears all comments.
    Reasoning: Building a secure, RESTful discussion API with proper authentication and data persistence would have required additional backend infrastructure (e.g., database schemas, user permissions) and more development time than allotted for this phase. By faking the posts, we could demonstrate the intended UX flow—input form, loading state, success notification—without risking incomplete or buggy server code.








  • Keyword Search Integration


    The A3 design envisioned each post tagged with multiple keywords (game titles, resource types) and a search interface returning filtered results by tag. Deviation: Because our “fake” posts lack a true tagging schema on the backend, we implemented a simplified search: the input field performs a client-side substring match only against the post title and body text.
    Reasoning: Proper keyword indexing and tag management depend on structured data stored server-side (e.g., a posts table with a many-to-many relationship to a keywords table). Without a real API endpoint to fetch tagged data, a full tag-based search would have produced inconsistent or misleading results. We chose reliable text matching now, with a plan to upgrade to API-driven tag search in future iterations.

Responsive Design And Dynamic Functionality

  • Mobile-First & Breakpoints
    Base styles in base.css target 320 px screens for default fonts and spacing. Media queries in layout.css and desktop3.css at 768 px and 1024 px adjust container width (90 vw → 70 vw), grid columns, and root variables.
  • Fluid Gallery & Post Grids
    The gallery (gallery.css) uses CSS Grid auto-fit minmax(200px, 1fr); resource cards (components.css) use Flexbox flex-wrap: wrap, ensuring items wrap and resize smoothly.
  • Accordion Animations
    In accordion.css, toggling .is-open animates max-height and padding over 0.3–0.4 s. accordion.js handles click events to add/remove this class without manual height calculations.
  • Slide-In Image Viewer
    Viewer styles (viewer.css) position the panel off-screen via transform: translateX(100%). viewer.js adds an .open class to slide it into view over 0.3 s and trap background scroll.
  • AJAX Data Loading & Form Submission
    getData.js fetches posts (GET) and renders cards; postFormData.js submits forms (POST); message.js shows success/error toasts, all without page reload.
  • Interactive Search Bar
    Search styles in site_map.css animate the input with keyframes. site_map.js listens for input events and filters rendered post cards in real time by matching titles/body text.
  • Custom Carousel Slider
    The carousel (style_guide.js) rotates slides every 5 s with opacity transitions from cyberpunk_layout.css. Prev/next controls and dot indicators toggle .active classes on hover and slide change.

Genai Mt Acknowledgement

I used OpenAI’s ChatGPT for generating technical descriptions and code examples, DeepL Translator (with Google Translate as a backup) for precise Chinese↔English translations, and carefully crafted prompts to guide each task. When ChatGPT’s outputs were too generic or omitted project-specific files, I refined prompts with explicit filenames and CSS variables and manually enriched the results; when DeepL mistranslated CSS terms like “auto-fit minmax,” I cross-checked with Google Translate, retained original terminology, and annotated key terms. To avoid overly long sections, I broke large tasks into smaller sub-sections and consolidated duplicate points. All AI-assisted text was thoroughly reviewed and rewritten to reflect my own analysis and ensure the document’s accuracy and originality. This process complied fully with the course’s GenAI & MT Usage Framework and upheld strict academic integrity.
