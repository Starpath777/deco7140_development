GenAI & Machine Translation Acknowledgement

Tools Used

  • OpenAI ChatGPT (text generation, code examples)
  • DeepL Translator (Chinese↔English translation, terminology checking)
  • Google Translate (secondary translation verification)

Usage Scenarios & Prompts

  • OpenAI ChatGPT
    • “Please describe the responsive design and dynamic functionality details based on the provided CSS/JS code.”
    • “Expand the ‘Variations from A3 Design Proposal’ section, explaining technical limitations and feasibility issues.”
  • DeepL Translator
    • “Translate the following Chinese technical text into clear English: …”
    • “Please proofread and polish this English segment into fluent Chinese technical documentation.”
  • Google Translate (used to cross-check DeepL results)

Challenges & Solutions (SEAL Format)

Challenge 1: Generic Outputs Not Matching Project Context

Situation

When using ChatGPT to generate implementation details, the model produced generic descriptions without cyberpunk flair or specific file references.

Evidence

AI outputs did not reference key files like visceral_design.css or cyberpunk_layout.css.

Action

  • Provided precise context in prompts, listing filenames and CSS variables.
  • Manually filtered and enriched AI results with relevant module details.

Learning

Accurate context and examples in prompts greatly improve AI output relevance and specificity.

Challenge 2: Inconsistent Translation of Technical Terms

Situation

DeepL sometimes mistranslated CSS terms like “auto-fit minmax,” causing technical inaccuracies.

Evidence

Translated text included unnatural phrases like “automatic fit minimum maximum range.”

Action

  • Compared results from DeepL and Google Translate, retaining original terminology with Chinese annotations.
  • Added a glossary of terms at the end of the document for clarity.

Learning

Cross-tool verification and manual proofreading ensure term consistency and accuracy.

Challenge 3: Overlong Sections from Bulk Expansion

Situation

Asking AI to expand the entire “Variations” section at once resulted in lengthy, unfocused text.

Evidence

Expanded content exceeded 500 words, making it hard to identify key points quickly.

Action

  • Broke the task into sub-tasks (e.g., “technical limitations,” “feasibility issues”).
  • Reviewed and consolidated similar points to tighten the structure.

Learning

Segmenting large tasks and iterative review improves clarity and readability.

Challenge 4: Maintaining Academic Integrity & Originality

Situation

Reliance on AI risked conflicting with the course’s requirement for independent critical thinking.

Evidence

Some AI-generated passages closely mirrored online examples, lacking personal insight.

Action

  • Manually reviewed and rewrote all AI-assisted content to include personal analysis and terminology.
  • Declared in the introduction that content was AI-assisted and fully revised for transparency.

Learning

Transparent disclosure and deep editing are essential to uphold academic standards.

Challenge 5: Generating Nine Game Section Layouts

Situation

AI initially produced incomplete forum layouts, missing some of the nine required game categories.

Evidence

Code outputs omitted categories like TCG and SPG.

Action

  • Specified all nine categories in the prompt and requested per-category templates.
  • Generated a single-column template first, then looped to create nine sections, finalizing manually.

Learning

Clear requirement lists and phased generation ensure completeness and accuracy.

Challenge 6: Inaccurate Resource Integration Descriptions

Situation

AI’s initial resource panel text lacked “upload,” “download,” and “preview” steps for mods and maps.

Evidence

Missing references to getData.js and postFormData.js API fields.

Action

  • Included resource type breakdown and operation steps in prompts.
  • Aligned examples with actual API parameter names for consistency.

Learning

Embedding API details in prompts yields outputs that match implementation needs.

Challenge 7: Misaligned Keyword Search Logic

Situation

AI recommended backend indexing and a tag system, but the project only implements front-end text matching.

Evidence

Provided pseudo-code assumed database and server APIs, unsuitable for the front end.

Action

  • Clarified that only front-end text.match() is supported in prompts.
  • Supplied specific site_map.js function and DOM examples.

Learning

Defining implementation boundaries guides AI toward executable solutions.

Challenge 8: Consistent Cyberpunk Styling

Situation

AI-generated CSS often omitted custom neon variables and spacing conventions.

Evidence

Outputs lacked --neon-color and --space-# usage.

Action

  • Emphasized root variables and spacing scale in prompts.
  • Generated layout then style modules separately, merging and tweaking manually.

Learning

Phased prompts and explicit variable mentions ensure stylistic consistency.

Integrity Statement

All Generative AI and Machine Translation tools used in this document comply with the course’s GenAI & MT Usage Framework. AI-assisted content has been thoroughly reviewed and revised to ensure accuracy, originality, and adherence to academic integrity.
