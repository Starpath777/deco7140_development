Accessibility Assessment

Perceivable: 1.4 Distinguishable

To ensure that all users can perceive content clearly, including those with visual impairments or color blindness, the design will follow several WCAG 2.1 Level AA guidelines under the "Distinguishable" category:

  • 1.4.1 Use of Color: Critical information such as status indicators or error messages will not rely on color alone. Instead, we will combine color with textual cues (e.g., labels like "Success" or "Error") and visual icons. This ensures that users with color vision deficiencies can still understand the message.
  • 1.4.3 Minimum Contrast: All text elements, especially body copy and interface labels, will maintain a minimum contrast ratio of 4.5:1 against their background. Larger headings will maintain at least 3:1. We will also implement a high-contrast mode option for users who require stronger visual separation.
  • 1.4.4 Resize Text: The website will use responsive units (like em, rem, %) for typography. This enables users to zoom text up to 200% without loss of structure, functionality, or content overlap, meeting responsive design best practices.

These adjustments aim to make visual content accessible to a wider range of users without sacrificing aesthetic quality.

Operable: 2.1.1 Keyboard Navigability

Some users rely entirely on keyboard navigation, such as individuals with motor impairments, users of screen readers, or those who avoid mouse input due to personal preferences. Our site design addresses this need by ensuring:

  • All interactive elements (like links, buttons, and form fields) are fully operable using a keyboard alone (e.g., via Tab, Enter, or Space).
  • We will use semantic HTML elements (like <button>, <nav>, <form>) to guarantee predictable tabbing and proper focus management.
  • Any custom widgets or dynamic UI components (such as modals, tabs, or accordions) will include JavaScript keyboard event listeners for accessibility.
  • A clearly visible "Skip to main content" link will appear at the top of the page for screen reader and keyboard users, allowing them to bypass repetitive navigation.

By providing keyboard accessibility from the ground up, we improve usability for a broad range of assistive technology users.

Understandable: 3.1.5 Reading Level

Accessibility is not only about interacting with content, but also understanding it. To support individuals with cognitive disabilities, reading challenges, or those for whom English is a second language, the following strategies will be used:

  • Every major content section will begin with a clear and concise summary, allowing users to grasp the key idea before diving into details.
  • We will incorporate visual aids such as icons, color coding, and diagrams to reinforce important points.
  • Written language will aim for a B1-B2 reading level (according to CEFR), using plain English, short sentences, and active voice to reduce confusion.
  • For those who need additional support, optional text-to-speech functionality will be considered, enabling auditory access to written content.

This approach ensures the content remains inclusive and understandable, supporting equitable access for all users.
