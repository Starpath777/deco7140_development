Accessibility Assessment

Perceivable: 1.4 Distinguishable

To ensure that all users can perceive content clearly, including those with visual impairments or color blindness, the design will follow several WCAG 2.1 Level AA guidelines under the "Distinguishable" category:

These adjustments aim to make visual content accessible to a wider range of users without sacrificing aesthetic quality.

1.4.1 Use of Color

Combine color with labels/icons so that colorblind users understand key messages.

1.4.3 Minimum Contrast

Maintain 4.5:1 contrast for text; enable high contrast mode.

1.4.4 Resize Text

Use relative units (em/rem) so text can scale up to 200% without breaking layout.

Operable: 2.1.1 Keyboard Navigability

Some users rely entirely on keyboard navigation, such as individuals with motor impairments, users of screen readers, or those who avoid mouse input due to personal preferences. Our site design addresses this need by ensuring:

By providing keyboard accessibility from the ground up, we improve usability for a broad range of assistive technology users.

Need: Keyboard-only Navigation

All links/buttons/forms should work with Tab, Enter, and Space.

Understandable: 3.1.5 Reading Level

Accessibility is not only about interacting with content, but also understanding it. To support individuals with cognitive disabilities, reading challenges, or those for whom English is a second language, the following strategies will be used:

This approach ensures the content remains inclusive and understandable, supporting equitable access for all users.

Use Clear Summaries

Start each section with a simple summary to aid understanding.

Add Visual Aids

Use icons, diagrams, and colors to reinforce key points.

Use Plain Language

Write at B1-B2 level using short sentences and active voice.

Add Audio Support

Optional text-to-speech for auditory learners or vision-impaired users.
